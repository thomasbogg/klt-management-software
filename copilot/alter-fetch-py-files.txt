Here is a little more on the new syntax:

    The self.TableX().Selection().Set(['column_x', 'column_y', 'column_z']) syntax is no loger valid. Instead use this:
        select = search.tableX.select()
        select.columnX()
        select.columnY()
        select = search.tableY.select()
        select.columnX()
        select.columnY()

    The self.TableX().Conditions().Set(column_x=True, column_y=('x', 'y'), has_column_z=) syntax is also obsolete. Change it to:
        where = search.tableX.where()
        where.columnX().isTrue()
        where.columnY().isIn(('x', 'y'))
        where.columnZ().isNotNullEmptyOrFalse() ## again, for more on available condition-setting consult column.py
